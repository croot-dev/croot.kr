## 정의


**Vertical SaaS**는 특정 산업이나 업종만을 위해 설계된 클라우드 소프트웨어 서비스다. 의료, 건설, 법률, 레스토랑 등 하나의 산업에 집중하여 해당 분야의 고유한 워크플로우, 규제 요건, 비즈니스 로직을 깊이 있게 다룬다.


반대 개념인 **Horizontal SaaS**는 산업과 무관하게 특정 기능(CRM, 협업, 마케팅 등)을 범용으로 제공한다.


| Vertical SaaS                              | Horizontal SaaS                                 |
| ------------------------------------------ | ----------------------------------------------- |
| - Veeva (제약)
- Procore (건설)
- Toast (레스토랑) | - Salesforce (CRM)
- Slack (협업)
- HubSpot (마케팅) |
| → 특정 산업에 특화
→ 좁고 깊은 기능                     | → 모든 산업에 적용
→ 넓고 얕은 기능                          |


---


## 핵심 특징


### 1. 산업 특화 기능


해당 업종의 전문 용어, 프로세스, 규제를 반영한다. 의료용 SaaS라면 HIPAA 규정 준수, 전자의무기록(EMR), 보험 청구 기능이 기본이다.


### 2. 올인원 솔루션


한 산업 내 필요한 여러 기능을 통합 제공한다. 레스토랑용 SaaS는 POS, 재고관리, 예약, 배달 연동을 하나의 플랫폼에서 처리한다.


### 3. 높은 전환 비용


업무 프로세스 깊숙이 통합되어 다른 솔루션으로 교체하기 어렵다. 결과적으로 고객 이탈률이 낮다.


### 4. 효율적인 고객 획득


산업 컨퍼런스, 협회, 전문 미디어 등 명확한 채널이 있고, 업계 내 입소문이 빠르다.

## 수익 모델


### 기본: 구독료 (Subscription)


월/연 단위 소프트웨어 사용료


### 확장: 임베디드 금융 (Embedded Finance)

- **결제 수수료**: 플랫폼 내 결제 처리 시 거래당 수수료
- **대출**: 축적된 거래 데이터 기반 소상공인 대출
- **보험**: 산업 특화 보험 상품 중개

### 기타

- 마켓플레이스 거래 수수료
- 데이터 분석/벤치마킹 서비스

## Horizontal SaaS와 비교


| 구분     | Horizontal SaaS | Vertical SaaS |
| ------ | --------------- | ------------- |
| 타겟     | 전 산업            | 특정 산업         |
| 기능     | 범용 (CRM, 협업 등)  | 산업 특화         |
| 시장 규모  | 큼               | 상대적으로 작음      |
| 경쟁 강도  | 높음              | 낮음 (틈새시장)     |
| 고객 이탈률 | 상대적으로 높음        | 낮음            |
| 진입 장벽  | 기술력             | 기술력 + 도메인 전문성 |


---


## 한계

- **TAM 제한**: 특정 산업만 대상이므로 시장 규모에 천장 존재
- **산업 의존성**: 타겟 산업 침체 시 직접적 영향
- **도메인 전문성 필요**: 해당 산업 경험자 없이는 제품 개발 어려움
- **레거시 시스템**: 전통 산업의 기존 시스템 교체 저항
